# 통장 대사

> **소요 시간**: 약 20분 | **난이도**: ★★★ | **주기**: 매월 (월말 마감 전)

시스템(장부)에 기록된 계좌 잔액과 실제 은행 통장 잔액이 일치하는지 확인하고, 차이가 있으면 원인을 파악하여 조정하는 프로세스입니다.

## 전체 흐름

```
1. 은행 거래내역 조회     (3분)  → 아파트뱅크에서 실제 거래 확인
2. 시스템 잔액 확인       (3분)  → 제예금명세서로 장부 잔액 확인
3. 잔액 비교             (2분)  → 통장 vs 장부 대사
4. 차이 원인 분석        (10분) → 미입력 전표, 미처리 수납 등 파악
5. 조정 처리             (2분)  → 누락 건 입력·수정
```

---

## 1단계: 은행 거래내역 조회 (3분)

> **메뉴 경로**: 아파트뱅크 > [은행거래내역](/bank/transaction) (BANK0010)

### 절차

1. 은행거래내역 화면 진입
2. **조회 기간** 설정 (해당 월 1일 ~ 말일)
3. **계좌 선택** (관리비 통장, 일반 통장 등)
4. **조회** 실행
5. 거래 목록 확인:
   - 거래일시
   - 적요(내용)
   - 출금액 / 입금액
   - 잔액
   - 수납처리 여부

### 최근 잔액 확인

화면 상단의 **최근잔액조회**에서 해당 계좌의 최신 잔액을 확인합니다. 이 잔액은 조회 기간과 무관하게 가장 최근 거래 기준입니다.

::: tip 계좌조회 로그
거래내역이 조회되지 않으면 **계좌조회로그**를 확인하세요. 거래 없음, 시스템 오류, 비밀번호 오류 등 원인을 파악할 수 있습니다.
:::

---

## 2단계: 시스템 잔액 확인 (3분)

### 제예금명세서

> **메뉴 경로**: 회계 > [전표관리](/acct/voucher) > 제예금명세서 (ACCT2100)

계좌별로 회계 장부에 기록된 잔액을 확인합니다.

### 총계정원장

> **메뉴 경로**: 회계 > [장부관리](/acct/ledger) > 총계정원장 (ACCT3010)

보통예금 계정의 일별 차변·대변·잔액을 확인합니다.

---

## 3단계: 잔액 비교 (2분)

아래 표를 작성하여 비교합니다.

| 계좌 | 통장 잔액 (은행) | 장부 잔액 (시스템) | 차이 |
|------|------------------|-------------------|------|
| 관리비 통장 | ○○○원 | ○○○원 | ○○원 |
| 일반 통장 | ○○○원 | ○○○원 | ○○원 |
| 기타 | ○○○원 | ○○○원 | ○○원 |

**차이가 0원이면** → 대사 완료, 5단계로 이동할 필요 없음

**차이가 있으면** → 4단계에서 원인 분석

---

## 4단계: 차이 원인 분석 (10분)

### 주요 원인과 확인 방법

| 원인 | 증상 | 확인 방법 |
|------|------|-----------|
| **미입력 지출 전표** | 통장 잔액 < 장부 잔액 | 통장 출금 내역 중 전표 미입력 건 찾기 |
| **미입력 수납** | 통장 잔액 > 장부 잔액 | 통장 입금 내역 중 수납 미처리 건 찾기 |
| **이체 시차** | 월말 이체가 다음 달에 반영 | 이체 요청일 vs 실제 출금일 비교 |
| **카드 결제 시차** | 승인일 ≠ 결제일 | 카드 매출전표 날짜 확인 |
| **이자 수입 미반영** | 소액 차이 | 통장 이자 내역 전표 입력 누락 |
| **수수료 미반영** | 소액 차이 | 이체 수수료, 카드 수수료 전표 입력 누락 |
| **가수금 미정리** | 입금 있지만 수납 미처리 | 가수금관리(RECP1051) 확인 |

### 분석 절차

1. 은행거래내역(BANK0010)에서 해당 월 거래를 **한 건씩** 확인
2. 각 거래의 **수납처리 여부** 칼럼 확인
   - 처리됨 → OK
   - 미처리 → 누락 건으로 표시
3. 전표관리(ACCT2010)에서 해당 월 전표 목록과 대조
4. 통장에는 있지만 전표에 없는 건 → **미입력 전표**
5. 전표에는 있지만 통장에 없는 건 → **이체 시차** 또는 **오류 전표**

---

## 5단계: 조정 처리 (2분)

차이 원인별로 조정합니다.

### 미입력 전표 추가

> **메뉴 경로**: 회계 > [전표관리](/acct/voucher) > 전표입력 (ACCT2010)

누락된 지출/수입 전표를 입력합니다. 상세 방법은 [지출 전표 입력](/guide/scenarios/expense-voucher)을 참고하세요.

### 미처리 수납 추가

> **메뉴 경로**: 수납 > [수납처리](/recp/process) (RECP1010)

미처리 입금 건을 수납 처리합니다. 상세 방법은 [수납 처리](/guide/scenarios/collection)를 참고하세요.

### 이체 시차 건

월말 이체 시차로 인한 차이는 별도 조정 없이 **비고로 기록**합니다. 다음 달 통장 대사 시 자연 해소됩니다.

## 자주 발생하는 문제

### 은행거래내역이 조회되지 않아요

**원인**: 공인인증서 만료, IP 변경, 은행 시스템 점검
**해결**: 계좌조회로그를 확인하세요. IP 변경으로 서비스가 중단된 경우 아파트뱅크 지원팀에 연락하세요.

### 소액 차이(수백 원)가 계속 남아요

**원인**: 이체 수수료, 통장 이자, 소액 잔돈 등
**해결**: 통장 거래내역에서 수수료·이자 내역을 확인하고, 해당 금액만큼 전표를 입력하세요.

### 매월 잔액이 안 맞아서 힘들어요

**원인**: 전표 입력이 수시로 누락되는 경우
**해결**: 지출이 발생할 때마다 **즉시** 전표를 입력하는 습관을 기르세요. 아파트뱅크 거래내역과 연동하면 자동으로 매칭할 수 있습니다.
