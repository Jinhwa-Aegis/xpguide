# 증빙 관리

> **소요 시간**: 약 10분 | **난이도**: ★★☆ | **주기**: 수시 (거래 발생 시)

아파트뱅크에 연동된 계좌에서 입금이 발생했을 때 수납 미등록 내역을 확인하고, 오류를 해소하여 모든 입금 건이 정상 수납 처리되도록 관리하는 프로세스입니다.

## 전체 흐름

```
1. 수납 후 미등록 내역 확인  (3분)  → 입금됐지만 수납 미처리된 건 조회
2. 입금금액 처리             (3분)  → 미처리 입금을 수납으로 연결
3. 수납결과 조회             (2분)  → 처리 결과 확인
4. 수납처리 오류 내역 처리   (3분)  → 오류 유형별 해소
5. 계좌조회 로그 확인        (수시) → 계좌 조회 문제 발생 시 원인 파악
```

---

## 사전 이해: 빠른계좌 서비스

::: tip 빠른계좌 서비스 사용 시
빠른계좌 서비스를 이용하는 단지는 **세대별 가상계좌**로 입금이 자동 처리됩니다. 입금 즉시 수납처리가 완료되고 아파트뱅크에 자동 반영됩니다. 이 경우 수동 처리 건수가 크게 줄어듭니다.
:::

---

## 1단계: 수납 후 미등록 내역 확인 (3분)

> **메뉴 경로**: 아파트뱅크 > [증빙내역](/bank/evidence) > 수납후미등록내역 (BANK0040)

입금은 되었지만 수납 시스템에 아직 등록되지 않은 건을 조회합니다.

### 절차

1. 수납후미등록내역 화면 진입
2. **조회 기간** 설정
3. **조회** 실행
4. 미등록 건 목록 확인:
   - 입금 일시
   - 입금 금액
   - 입금 계좌
   - 입금인 (적요)
   - 처리 상태

### 확인 포인트

- [ ] 오늘 날짜 포함 최근 미등록 건이 있는가?
- [ ] 장기 미처리 건(3일 이상)이 있는가?
- [ ] 금액이 관리비와 일치하지 않는 이상 입금 건이 있는가?

---

## 2단계: 입금금액 처리 (3분)

> **메뉴 경로**: 아파트뱅크 > [증빙내역](/bank/evidence) > 입금금액처리 (BANK0050)

미처리 입금 건을 해당 세대의 수납으로 연결합니다.

### 절차

1. 입금금액처리 화면 진입
2. 처리할 미등록 입금 건 선택
3. 입금 내용(적요, 금액)을 기준으로 **해당 세대** 확인
4. 세대 동·호수 매칭 후 **수납 처리** 실행
5. 처리 완료 확인

### 입금 관리자 팝업 활용

> **메뉴 경로**: 아파트뱅크 > [증빙내역](/bank/evidence) > 입금관리자팝업 (BANK0060)

입금 건의 세대 식별이 어려울 때 팝업에서 입금 내역 상세 정보를 조회하고 수납 세대를 수동으로 지정합니다.

---

## 3단계: 수납결과 조회 (2분)

처리한 입금 건이 정상적으로 수납되었는지 확인합니다.

> **메뉴 경로**: 수납 > [수납조회](/recp/inquiry)

1. 처리 일자 기준으로 수납 내역 조회
2. 방금 처리한 세대·금액이 수납 목록에 있는지 확인
3. 이상이 없으면 완료

---

## 4단계: 수납처리 오류 내역 처리 (3분)

> **메뉴 경로**: 아파트뱅크 > [증빙내역](/bank/evidence) > 수납처리오류내역 (BANK0090)

자동 수납 처리 중 오류가 발생한 건을 확인하고 해소합니다.

### 오류 유형별 해결 방법

| 오류 유형 | 원인 | 해결 방법 |
|-----------|------|-----------|
| **수납동수없음** | 입금 세대를 특정할 수 없음 | 입금인(적요)으로 세대 확인 후 수동 수납 |
| **금액불일치** | 입금액이 고지 금액과 다름 | 고지 금액 확인 후 초과분 처리 또는 세대 확인 |
| **납부자미확인** | 세대 정보가 없거나 검색 불가 | 입금인 연락처 확인 또는 창구 내방 안내 |
| **계좌미등록** | 입금 계좌가 단지 계좌에 미등록 | 단지 수납 계좌 등록 상태 확인 |

### 오류 건 수동 처리 절차

1. 수납처리오류내역에서 오류 건 확인
2. 오류 유형 파악 후 위 표 기준으로 조치
3. 세대 확인 완료 후 [수납처리](/recp/process)(RECP1010)에서 수동 수납
4. 처리 결과 재확인

::: warning 미처리 오류 건 방치 금지
수납처리 오류 건을 방치하면 해당 세대가 미납 상태로 남아 미수금 관리나 고지서 발행에 혼선이 생깁니다. 발생 당일 처리를 원칙으로 하세요.
:::

---

## 5단계: 계좌조회 로그 확인 (수시)

> **메뉴 경로**: 아파트뱅크 > [증빙내역](/bank/evidence) > 계좌조회로그 (BANK0080)

은행 거래내역이 조회되지 않거나 연동 오류가 발생할 때 원인을 파악합니다.

### 절차

1. 계좌조회로그 화면 진입
2. 문제 발생 일시 기준으로 조회
3. 로그 메시지로 원인 파악

### 주요 로그 유형

| 로그 유형 | 의미 | 조치 |
|-----------|------|------|
| 거래 없음 | 해당 기간 입출금 없음 | 정상 상태 (조치 불필요) |
| 인증 오류 | 공인인증서 만료 또는 오류 | 공인인증서 갱신 후 재시도 |
| IP 차단 | 접속 IP 변경으로 서비스 차단 | 아파트뱅크 고객센터에 IP 변경 등록 요청 |
| 은행 점검 | 은행 시스템 점검 중 | 점검 종료 후 재조회 |

## 자주 발생하는 문제

### 입금 건인데 수납이 안 되어 있어요

**원인**: 입금인(적요)이 세대와 일치하지 않아 자동 매칭에 실패한 경우입니다.
**해결**: 수납후미등록내역(BANK0040)에서 해당 입금 건을 찾아, 입금관리자팝업(BANK0060)으로 세대를 수동 확인한 뒤 입금금액처리(BANK0050)로 연결하세요.

### 은행 거래내역이 아파트뱅크에 안 보여요

**원인**: 공인인증서 만료, IP 변경, 은행 시스템 점검 등이 원인입니다.
**해결**: 계좌조회로그(BANK0080)에서 오류 메시지를 확인하세요. IP 변경이 원인이라면 아파트뱅크 고객센터에 새 IP 등록을 요청해야 합니다.

### 금액불일치 오류 건은 어떻게 처리하나요?

**원인**: 세대가 고지 금액과 다른 금액을 입금한 경우입니다.
**해결**: 고지 금액보다 많이 납부한 경우 초과분을 선납으로 처리하거나 다음 달 차감합니다. 적게 납부한 경우 부족분을 미납으로 남기고 세대에 통보하세요.

### 계속 오류 건이 발생하는 특정 세대가 있어요

**원인**: 세대 정보(이름, 연락처 등)와 입금 적요가 반복적으로 불일치하는 경우입니다.
**해결**: 해당 세대에 납부 시 **세대번호(동·호수) 또는 납부자 성명을 적요에 정확히 기재**하도록 안내하세요. 자동이체 전환을 권장하는 것도 좋습니다.
