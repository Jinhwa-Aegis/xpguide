# 단지 초기 설정

> **소요 시간**: 약 3시간 | **난이도**: ★★★ | **주기**: 최초 1회 (신규 단지)

XpERP를 처음 사용하는 단지에서 시스템 운영에 필요한 기초 정보를 등록하는 프로세스입니다. 이 설정이 모든 업무의 토대가 됩니다.

## 전체 흐름

```
1.  단지정보등록         (15분) → 단지코드·명칭·주소·사업자 정보
2.  동호생성             (30분) → 동별 라인·호수·층수·면적 구성
3.  공동동호관리         (10분) → 임의 동호 추가 (선택)
4.  세대검침환경등록     (20분) → 세대별 검침 항목 설정
5.  환경설정             (30분) → 검침·부과·고지서·할인 환경
6.  조견표 등록          (20분) → 관리비·주차비 조견표
7.  입주자단체등록       (10분) → 입주자대표회의 등 단체 정보
8.  결재라인등록         (15분) → 장표별 결재 경로 설정
9.  연말정산기본정보등록 (10분) → 급여 관련 공제 정보
10. 모바일발송 설정      (15분) → 발신번호 인증·SMS/카카오 연동
```

---

::: danger 단지코드는 수정 불가
단지코드는 **최초 등록 후 절대 수정할 수 없습니다.** 단지코드를 잘못 등록하면 모든 정보를 처음부터 다시 등록해야 합니다. 입력 전에 반드시 확인하세요.
:::

---

## 1단계: 단지정보등록 (15분)

> **메뉴 경로**: 단지관리 > [단지정보](/dang/dang-info) > 단지정보등록 (DANG0010)

단지의 기본 정보를 등록합니다. 모든 업무의 기준이 되는 첫 번째 설정입니다.

### 절차

1. 단지정보등록 화면 진입
2. 다음 항목 입력:

| 항목 | 설명 | 주의 |
|------|------|------|
| **단지코드** | 시스템 내 고유 식별 코드 | **등록 후 수정 불가** |
| 단지명 | 아파트 공식 명칭 | |
| 주소 | 단지 도로명 주소 | |
| 세대수 | 총 세대 수 | |
| 사업자등록번호 | 관리소 사업자등록번호 | |
| 대표자명 | 관리소장 성명 | |
| 업태/종목 | 부동산 / 아파트관리 | |

3. **저장**

::: warning 사업자 정보 정확히 입력
사업자등록번호·대표자명 등은 세금계산서, 원천징수 신고 등에 사용됩니다. 관할 세무서에 등록된 정보와 정확히 일치해야 합니다.
:::

---

## 2단계: 동호생성 (30분)

> **메뉴 경로**: 단지관리 > [단지정보](/dang/dang-info) > 동호생성 (DANG0030)

단지의 동·호수 구조를 생성합니다. 검침, 부과, 입주자 관리 등 모든 세대별 업무의 기준이 됩니다.

### 절차

1. 동호생성 화면 진입
2. 동 정보 입력:
   - 동 번호 (예: 101동, 102동)
   - 지하 층수
   - 지상 층수
   - 층당 세대 수 (라인 수)
3. **동라인생성** 버튼 클릭:
   - 승강기 라인 지정
   - 라인별 순서 설정 (호수 부여 기준)
4. 면적 정보 입력:
   - 세대별 전용면적
   - 공용면적 (비례 배분 기준)
5. 생성 결과 확인 후 **저장**

### 확인 포인트

- [ ] 동 수와 실제 건물 수가 일치하는가?
- [ ] 각 동의 층수가 정확한가?
- [ ] 호수 부여 방식이 실제 단지와 동일한가? (예: 라인별 순서)
- [ ] 면적 정보가 건축물 대장과 일치하는가?

::: warning 동·호수 변경 제한
동·호수 정보는 입주자 정보와 연결됩니다. 등록 후 변경하려면 해당 세대의 **전출 처리를 완료한 후**에만 삭제·수정할 수 있습니다. 초기 등록 시 신중하게 입력하세요.
:::

## 3단계: 공동동호관리 (10분, 선택)

> **메뉴 경로**: 단지관리 > [단지정보](/dang/dang-info) > 공동동호관리 (DANG0036)

관리사무소, 경비실, 노인정 등 일반적인 동호 생성으로 만들 수 없는 임의 동호를 추가 등록합니다.

### 사용 예시

| 용도 | 임의 동호 예시 |
|------|----------------|
| 관리사무소 | 000동 000호 |
| 경비실 | 경비동 01호 |
| 공용 시설 | 노인정, 커뮤니티센터 |
| 상가 | 상가동 101호 |

::: tip 공동동호 활용
공동동호를 등록하면 관리사무소·상가 등에도 관리비 부과 및 검침 관리가 가능합니다.
:::

## 4단계: 세대검침환경등록 (20분)

> **메뉴 경로**: 단지관리 > [단지정보](/dang/dang-info) > 세대검침환경등록 (DANG0035)

세대별로 검침이 필요한 항목(전기, 수도, 온수, 가스, 난방 등)을 설정합니다.

### 절차

1. 세대검침환경등록 화면 진입
2. 세대별로 각 검침 항목 설정:

| 검침 항목 | 설정값 | 설명 |
|-----------|--------|------|
| 전기 | Y / N | 개별 전기계량기 보유 여부 |
| 수도 | Y / N | 개별 수도계량기 보유 여부 |
| 온수 | Y / N | 온수 계량기 보유 여부 |
| 가스 | Y / N | 개별 가스계량기 보유 여부 |
| 난방 | Y / N | 개별 난방계량기 보유 여부 |
| 정수 | Y / N | 정수기 계량기 보유 여부 |

3. **계량기 자리수** 설정 (계량기 표시 최대 자릿수)
4. 저장

::: danger 검침환경 누락 주의
특정 세대에 검침환경이 등록되지 않으면 해당 검침 항목이 **검침 화면에 표시되지 않습니다.** 검침 누락이 발생하면 관리비 부과 오류로 이어집니다.
:::

### 대량 설정 방법

세대 수가 많은 경우 엑셀 업로드로 일괄 등록할 수 있습니다.
- 엑셀 양식 다운로드 → 항목 입력 → 업로드 실행
- 파일 형식: **Excel 97-2003(.xls)**

## 5단계: 환경설정 (30분)

> **메뉴 경로**: 단지관리 > [환경설정](/dang/dang-env)

단지 운영에 필요한 각종 환경을 설정합니다.

### 검침환경 (DANG-ENV-검침)

| 설정 항목 | 내용 |
|-----------|------|
| 검침 주기 | 월 검침 또는 격월 검침 |
| 검침 방법 | 직접 검침 / 원격 검침 |
| 검침일 | 월 검침 기준일 |

### 부과환경 (DANG-ENV-부과)

| 설정 항목 | 내용 |
|-----------|------|
| 부과 기준 | 면적 기준 / 세대 기준 / 혼합 |
| 납부 기한 | 당월 말 / 익월 말 등 |
| 연체 가산금 | 연체료율 및 적용 기준 |

### 고지서인쇄환경

| 설정 항목 | 내용 |
|-----------|------|
| 고지서 양식 | 단지 맞춤 양식 선택 |
| 인쇄 항목 | 고지서에 표시할 항목 선택 |
| 은행 가상계좌 | 수납 계좌 정보 |

### 단지검침환경

단지 전체의 공용 검침 방식 및 기준을 설정합니다.

### 할인항목

경로 우대, 장애인 할인 등 관리비 감면 항목을 설정합니다.

| 할인 유형 | 설명 |
|-----------|------|
| 경로우대 | 만 65세 이상 세대주 |
| 장애인 | 장애 등급에 따라 적용 |
| 국가유공자 | 관련 법령 기준 |

## 6단계: 조견표 등록 (20분)

> **메뉴 경로**: 단지관리 > [조견표](/dang/dang-table)

관리비 및 주차비 산정에 사용하는 단가 기준표를 등록합니다.

### 관리비 조견표

검침량에 따른 요금 계산 기준을 등록합니다.

| 항목 | 설정 내용 |
|------|-----------|
| 전기 요금 | 구간별 단가 (한전 기준) |
| 수도 요금 | 구간별 단가 (지자체 기준) |
| 가스 요금 | 구간별 단가 (도시가스 기준) |
| 난방 요금 | 구간별 단가 |

::: tip 요금 기준 변경 시
공공요금 인상 등으로 단가가 변경될 때 조견표를 업데이트하세요. 변경 전·후 내역이 모두 기록됩니다.
:::

### 주차비 조견표

차량 등록 수, 주차 구역, 시간 기준 등에 따른 주차비 단가를 설정합니다.

| 구분 | 예시 |
|------|------|
| 1대 등록 세대 | 기본 주차비 |
| 2대 이상 등록 | 추가 차량 할증 |
| 미등록 차량 | 일일 주차료 |

## 7단계: 입주자단체등록 (10분)

> **메뉴 경로**: 단지관리 > [단지정보](/dang/dang-info) > 입주자단체등록 (DANG0020)

단지 내 입주자 관련 단체를 등록합니다.

### 주요 등록 단체

| 단체명 | 설명 |
|--------|------|
| 입주자대표회의 | 단지 최고 의결 기관 |
| 선거관리위원회 | 대표회의 선거 관리 |
| 자생단체 | 노인회, 부녀회 등 |

### 입력 항목

- 단체명
- 구성원 정보 (대표자, 임원)
- 연락처
- 임기

## 8단계: 결재라인등록 (15분)

> **메뉴 경로**: 단지관리 > [단지정보](/dang/dang-info) > 결재라인등록 (DANG0051)

전표, 공문, 지출결의서 등 각 장표별 결재 경로를 설정합니다.

### 절차

1. 결재라인등록 화면 진입
2. 장표 종류 선택 (예: 지출결의서, 공문, 이체결의서)
3. 결재 단계 설정:
   - 기안자 → 검토자 → 결재자 순서로 지정
   - 대리 결재자 지정 (부재 시)
4. 저장

### 설정 예시

| 장표 | 결재 단계 |
|------|-----------|
| 지출결의서 | 담당 → 팀장 → 관리소장 |
| 이체결의서 | 담당 → 관리소장 → 입대의 대표 |
| 공문 | 담당 → 관리소장 |

::: tip 결재라인 미설정 시
결재라인이 등록되지 않은 장표는 결재 요청이 불가합니다. 아파트뱅크 이체 등 결재가 필요한 업무 전에 반드시 설정하세요.
:::

## 9단계: 연말정산기본정보등록 (10분)

> **메뉴 경로**: 단지관리 > [단지정보](/dang/dang-info) > 연말정산기본정보등록 (DANG0170)

직원 급여의 연말정산에 필요한 기본 정보를 등록합니다.

### 입력 항목

| 항목 | 설명 |
|------|------|
| 귀속 연도 | 정산 기준 연도 |
| 사업자등록번호 | 관리소 사업자번호 |
| 세액 계산 기준 | 간이세액표 기준 설정 |
| 비과세 항목 | 식대, 차량유지비 등 비과세 한도 |

::: tip 연도별 갱신
연말정산기본정보는 **매년 초** 해당 귀속 연도로 갱신해야 합니다. 세법 개정에 따라 비과세 한도 등이 변경될 수 있으니 국세청 기준을 확인하세요.
:::

## 10단계: 모바일발송 설정 (15분)

고지서·안내문 등을 SMS 또는 카카오 알림톡으로 발송하기 위한 설정입니다.

### 발신번호 인증 절차

모바일 발송을 사용하려면 **발신번호 인증**이 필수입니다. 아래 순서대로 진행하세요.

```
1. 통신가입증명서 발급  → 관리소 명의 통신사 방문 발급
2. 발신번호 대리인      → 대리인 신청서 작성 (관리소장 서명)
   신청서 작성
3. 서류 전산업체 제출   → 담당 전산업체에 서류 이메일/팩스 발송
4. 등록 완료 확인       → 업체로부터 등록 완료 통보 후 사용 가능
```

::: warning 발신번호 미인증 시 발송 불가
통신법 개정으로 인증되지 않은 번호로는 단체 문자 발송이 불가합니다. 반드시 인증 절차를 완료한 후 발송하세요.
:::

### SMS / 카카오 환경 설정

| 항목 | 설명 |
|------|------|
| 발신번호 | 인증 완료된 전화번호 입력 |
| SMS 기본 메시지 | 고지서 발송 기본 문구 설정 |
| 카카오 채널 | 카카오 비즈니스 채널 연동 |
| 발송 시점 | 자동 발송 기준 설정 |

## 초기 설정 완료 체크리스트

모든 단계를 완료한 후 아래를 점검하세요.

### 기본 구조

- [ ] 단지코드가 올바르게 등록되었는가?
- [ ] 모든 동·호수가 정확하게 생성되었는가?
- [ ] 세대별 면적 정보가 건축물 대장과 일치하는가?
- [ ] 각 세대의 검침 항목이 모두 설정되었는가?

### 환경 및 단가

- [ ] 검침·부과·고지서 환경이 설정되었는가?
- [ ] 관리비·주차비 조견표가 등록되었는가?
- [ ] 할인 항목이 설정되었는가?

### 운영 준비

- [ ] 입주자대표회의 정보가 등록되었는가?
- [ ] 결재라인이 모든 장표에 설정되었는가?
- [ ] 연말정산기본정보가 등록되었는가?
- [ ] 모바일 발신번호 인증이 완료되었는가?

## 자주 발생하는 문제

### 단지코드를 잘못 입력했어요

**원인**: 단지코드는 등록 후 수정이 불가합니다.
**해결**: 모든 데이터를 삭제하고 처음부터 재등록해야 합니다. **저장 전에 반드시 재확인**하세요. 의문이 있으면 전산업체에 문의하세요.

### 동호를 생성했는데 검침 화면에 세대가 안 나와요

**원인**: 세대검침환경등록(DANG0035)에서 해당 세대의 검침 항목이 설정되지 않은 경우
**해결**: 세대검침환경등록 화면에서 해당 세대를 찾아 필요한 검침 항목을 Y로 설정하세요.

### 면적을 잘못 입력했는데 수정할 수 있나요?

**원인**: 입주자가 없는 세대(미입주)는 수정 가능하지만, 입주 처리된 세대는 전출 처리 후에만 변경할 수 있습니다.
**해결**: 입주 전이라면 즉시 수정하세요. 입주 후라면 전출 처리 → 면적 수정 → 재입주 처리 순서로 진행하세요.

### 고지서 발송 시 SMS가 전송되지 않아요

**원인**: 발신번호 인증이 완료되지 않았거나 환경설정이 누락된 경우
**해결**: 발신번호 인증 절차(통신가입증명서 → 대리인신청서 → 전산업체 제출)가 완료되었는지 확인하세요. 완료 후에도 안 되면 전산업체에 문의하세요.

### 결재 요청이 안 돼요

**원인**: 해당 장표의 결재라인이 등록되지 않은 경우
**해결**: 결재라인등록(DANG0051) 화면에서 해당 장표의 결재 경로를 등록하세요.
