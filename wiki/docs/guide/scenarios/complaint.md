# 민원 접수/처리

> **소요 시간**: 건당 약 10분 | **난이도**: ★★☆ | **주기**: 수시 (민원 발생 시)

세대 또는 입주자의 시설 고장, 불편 사항 등 민원을 접수하고 작업을 지시·처리·완료 보고하는 프로세스입니다.

## 전체 흐름

### 사전 설정 (최초 1회)

```
1. 작업코드등록   (10분) → 자재/기기/시설물/기타 대분류 설정
2. 기기등록       (10분) → 전기·기계·조경 등 관리 기기 등록
3. 제품등록       (10분) → 자재 부품·규격·단가·적정재고량 등록
4. 거래처등록     (5분)  → 자재 구매처 등록
5. 민원코드등록   (10분) → 민원 대분류/소분류 코드 설정
```

### 일상 업무 (건당)

```
1. 민원 접수      (3분)  → 동호수·접수내용·민원코드 입력
2. 작업 처리      (5분)  → 작업내역 입력, 자재 출고 시 출고등록
3. 완료 보고      (2분)  → 처리결과 저장
4. 해피콜 서비스  (선택) → 만족도 확인
```

### 자재 관리 (수시)

```
1. 입고등록   → 거래처·부품·수량·단가 입력
2. 출고등록   → 사용부서·부품·수량 입력
3. 이월작업   → 월말 결산 전 처리
```

---

## 사전 설정 (최초 1회)

처음 시스템을 사용할 때 한 번만 설정합니다. 이후에는 변경이 있을 때만 수정합니다.

### 작업코드등록

> **메뉴 경로**: 민원 > [민원관리](/jobm/manage) > 작업코드등록 (JOBM1100)

민원 작업의 분류 체계를 설정합니다.

| 대분류 | 예시 소분류 |
|--------|-------------|
| 자재 | 배관부품, 전기자재, 소모품 |
| 기기 | 엘리베이터, 보일러, 소방설비 |
| 시설물 | 공용복도, 주차장, 조경 |
| 기타 | 민원상담, 행정업무 |

::: warning 작업코드 수정 불가
저장된 작업코드는 **수정이 불가**합니다. 코드명만 변경할 수 있으므로, 등록 전에 분류 체계를 신중하게 설계하세요.
:::

### 기기등록

> **메뉴 경로**: 민원 > [민원관리](/jobm/manage) > 기기등록 (JOBM1100)

단지 내 관리 대상 기기를 등록합니다.

- 전기설비: 수변전설비, 비상발전기, 소방전기
- 기계설비: 급수펌프, 오수펌프, 보일러
- 조경: 자동급수, 분수대

### 제품등록

> **메뉴 경로**: 민원 > [민원관리](/jobm/manage) > 제품등록 (JOBM1100)

민원 처리에 사용하는 자재 부품을 등록합니다.

| 입력 항목 | 설명 |
|-----------|------|
| 품명 | 부품 이름 |
| 규격 | 크기·형번 등 |
| 단가 | 구매 단가 |
| 적정재고량 | 재고 부족 알림 기준 수량 |

### 거래처등록

> **메뉴 경로**: 민원 > [민원관리](/jobm/manage) > 거래처등록 (JOBM1100)

자재 납품 업체 정보를 등록합니다. 입고 처리 시 사용됩니다.

### 민원코드등록

> **메뉴 경로**: 민원 > [민원관리](/jobm/manage) > 민원코드등록 (JOBM1200)

민원 종류를 대분류/소분류로 구분하여 등록합니다.

| 대분류 예시 | 소분류 예시 |
|-------------|-------------|
| 전기 | 세대 내 누전, 공용 조명 불량 |
| 수도 | 누수, 수압 불량 |
| 난방/온수 | 난방 불량, 온수 미공급 |
| 주차 | 장애물, 불법 주차 |
| 소음 | 층간소음, 공사 소음 |

## 1단계: 민원 접수 (3분)

> **메뉴 경로**: 민원 > [민원관리](/jobm/manage) > 세대민원접수및작업 (JOBM1200)

### 절차

1. 세대민원접수및작업 화면 진입
2. **행추가** 클릭
3. 다음 항목을 입력:

| 항목 | 설명 |
|------|------|
| 접수부서 | 담당 부서 선택 |
| 접수자 | 접수 직원 선택 |
| 동/호수 | 민원 세대 선택 |
| 접수일시 | 자동 입력 (수정 가능) |
| 민원코드 | 대분류→소분류 순서로 선택 |
| 접수내용 | 민원 내용 상세 입력 |

4. **저장** 클릭
5. 필요 시 **민원접수증** 출력 (입주자 교부용)

::: tip 민원접수증 출력
접수 후 입주자에게 접수증을 제공하면 처리 상황 문의 시 접수번호로 조회할 수 있습니다.
:::

---

## 2단계: 작업 처리 (5분)

> **메뉴 경로**: 민원 > [민원관리](/jobm/manage) > 세대민원접수및작업 (JOBM1200)

접수된 민원을 선택하여 작업 내역을 입력합니다.

### 절차

1. 접수된 민원 선택
2. **작업지시서** 출력 (선택) → 담당 기사에게 교부
3. 작업 완료 후 **작업내역 탭** 입력:
   - 작업일시
   - 담당자
   - 작업내용 (수행한 조치 상세 기술)
   - 사용 자재 (출고 연계)
4. 자재를 사용한 경우 → **출고등록** 연계 처리 (아래 참고)
5. **저장**

### 출고등록 (자재 사용 시)

> **메뉴 경로**: 민원 > [민원관리](/jobm/manage) > 출고등록 (JOBM1300)

민원 처리 중 창고 자재를 사용한 경우 출고를 등록합니다.

1. 출고등록 화면 진입
2. **사용부서** 선택
3. 사용 **부품 및 수량** 입력
4. **저장**

::: warning 출고 수량 초과 불가
현재 입고 수량을 초과하여 출고할 수 없습니다. 재고가 부족한 경우 먼저 입고등록을 처리하세요.
:::

---

## 3단계: 완료 보고 (2분)

> **메뉴 경로**: 민원 > [민원관리](/jobm/manage) > 세대민원접수및작업 (JOBM1200)

작업 완료 후 처리 결과를 저장하고 민원을 종결합니다.

### 절차

1. 해당 민원 선택
2. **처리상태**를 '완료'로 변경
3. 처리 결과 내용 최종 확인
4. **저장**
5. **세대민원처리대장** 출력 (월별 보고용)

---

## 4단계: 해피콜 서비스 (선택)

민원 처리 후 세대 입주자에게 연락하여 처리 만족도를 확인합니다.

- 전화 또는 문자로 처리 결과 안내
- 불만족 시 재처리 접수
- 만족도 점수는 시스템에 기록 가능

## 자재 관리

### 입고등록

> **메뉴 경로**: 민원 > [민원관리](/jobm/manage) > 입고등록 (JOBM1300)

자재를 구매하여 창고에 입고할 때 등록합니다.

1. 거래처 선택
2. 입고 부품·수량·단가 입력
3. 저장 → 재고 자동 증가

### 이월작업

> **메뉴 경로**: 민원 > [민원관리](/jobm/manage) > 이월작업 (JOBM1300)

월말 결산 전, 미완료 작업을 다음 달로 이월합니다.

::: danger 이월작업 시점
이월작업은 **월말 결산 전**에 반드시 처리해야 합니다. 결산 후에는 이월 처리가 제한될 수 있습니다.
:::

### 기기보수

> **메뉴 경로**: 민원 > [민원관리](/jobm/manage) > 기기보수 (JOBM1300)

단지 내 기기의 보수 이력을 기록합니다.

| 입력 항목 | 설명 |
|-----------|------|
| 기기 | 보수 대상 기기 선택 |
| 보수일자 | 실제 보수한 날짜 |
| 고장원인 | 고장 내용 상세 기술 |
| 처리내용 | 수행한 보수 작업 내용 |
| 담당자 | 처리 기사 |

## 민원 현황 조회

> **메뉴 경로**: 민원 > [민원조회](/jobm/inquiry) (JOBM1200)

### 주요 조회 기능

- **기간별 민원 목록**: 접수일 기준 조회
- **처리 상태별 조회**: 접수 중 / 처리 중 / 완료 / 보류
- **세대별 민원 이력**: 특정 동·호수의 민원 내역 확인
- **담당자별 처리 현황**: 직원별 처리 건수 파악

### 출력 서류

| 서류 | 용도 |
|------|------|
| 민원접수증 | 입주자 교부 |
| 작업지시서 | 담당 기사 교부 |
| 세대민원처리대장 | 월별 업무 보고 |

## 자주 발생하는 문제

### 민원코드가 목록에 없어요

**원인**: 민원코드가 등록되지 않았습니다.
**해결**: 민원코드등록(JOBM1200) 화면에서 해당 분류를 추가하세요. 대분류를 먼저 등록한 후 소분류를 추가해야 합니다.

### 출고 처리 시 재고 부족 오류가 나요

**원인**: 등록된 재고 수량보다 많은 양을 출고하려 하는 경우
**해결**: 입고등록(JOBM1300)에서 재고를 먼저 추가하거나, 출고 수량을 실제 재고 범위 내로 수정하세요.

### 작업코드를 잘못 등록했어요

**원인**: 작업코드는 저장 후 수정이 불가합니다.
**해결**: 코드명만 수정할 수 있습니다. 분류 구조 자체가 잘못된 경우 새 코드를 추가하고 기존 코드는 미사용 상태로 관리하세요.

### 이월작업을 깜빡하고 결산을 했어요

**원인**: 월말 결산 전 이월 처리를 하지 않은 경우
**해결**: 관리자에게 문의하여 결산 해제 후 이월 처리를 진행하세요. 다음 달부터는 **결산 전 이월작업**을 체크리스트에 포함하세요.

### 민원 처리 이력이 조회되지 않아요

**원인**: 조회 기간 또는 상태 필터 설정 오류
**해결**: [민원조회](/jobm/inquiry)에서 기간을 넓게 설정하고 처리 상태를 '전체'로 변경하여 다시 조회하세요.
