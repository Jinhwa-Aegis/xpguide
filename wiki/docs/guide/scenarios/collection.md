# 수납 처리

> **소요 시간**: 약 20분 | **난이도**: ★★☆ | **주기**: 매월 4주차 ~ 익월 1주차

부과된 관리비를 세대별로 수납 처리하고, 회계 전표와 연계하는 전체 프로세스입니다.

## 전체 흐름

```
1. 자동이체 결과 처리   (5분)  → 은행 자동이체 결과 반영
2. 개별 수납 처리       (5분)  → 창구·계좌이체 등 개별 건 입력
3. 오류 건 처리         (5분)  → 자동수납 오류·미매칭 건 정리
4. 수납 전표 연계       (3분)  → 회계 전표 자동 생성
5. 수납 결과 확인       (2분)  → 일보·집계표로 최종 확인
```

---

## 1단계: 자동이체 결과 처리 (5분)

> **메뉴 경로**: 수납 > [자동이체수납](/recp/auto-transfer)

은행에서 받은 자동이체 결과 파일을 시스템에 반영합니다.

### 절차

1. 은행에서 자동이체 결과 파일 수신
2. 자동이체수납 화면 진입
3. 결과 파일 **업로드**
4. 처리 결과 확인:
   - **정상 수납**: 자동 처리 완료
   - **출금 실패**: 잔액 부족 등 → 미납으로 분류
   - **이중 출금**: 중복 건 확인 필요
5. **저장** 클릭

### 체크리스트

- [ ] 자동이체 결과 파일을 은행에서 수신했는가?
- [ ] 정상 수납 건수와 금액이 예상과 일치하는가?
- [ ] 이중수납(중복 출금) 건이 없는가?

---

## 2단계: 개별 수납 처리 (5분)

> **메뉴 경로**: 수납 > [수납처리](/recp/process) > 일반수납 (RECP1010)

창구 납부, 계좌이체 등 자동이체 외 수납 건을 처리합니다.

### 절차

1. 수납처리 화면 진입
2. **수납일자**와 **수납기간** 입력
3. 해당 세대(동·호) 검색
4. 미납 내역 확인 후 수납 항목 선택
5. 수납 금액 입력
6. **처리** 버튼 클릭

### 부분 수납이 필요한 경우

세대에서 관리비 전액이 아닌 일부만 납부한 경우:

1. **부분수납** 모드 선택
2. 두 가지 방식 중 선택:
   - **일반 부분수납**: 체납분 우선 적용 후 나머지 이월
   - **미납 선택 부분수납**: 특정 미납 건을 선택하여 적용
3. 금액 입력 후 처리

::: tip 수납 취소
잘못 처리한 수납은 **수납취소** 기능으로 되돌릴 수 있습니다. 수납처리 화면에서 해당 건을 찾아 선택 후 취소를 실행하세요.
:::

## 3단계: 오류 건 처리 (5분)

> **메뉴 경로**: 수납 > [수납처리](/recp/process) > 자동수납오류처리 (RECP1020)

자동이체에서 매칭되지 않거나 오류가 발생한 건을 수동으로 정리합니다.

### 오류 유형별 처리

| 오류 유형 | 원인 | 처리 방법 |
|-----------|------|-----------|
| 세대 미매칭 | 입금자명과 세대정보 불일치 | 수동으로 세대 찾아서 매칭 |
| 금액 불일치 | 납부 금액 ≠ 부과 금액 | 부분수납 처리 또는 차액 확인 |
| 정상 입금인데 미처리 | 시스템 자동 매칭 실패 | 수동 수납 처리 |
| 가수금 발생 | 초과 납부 또는 미확인 입금 | 가수금관리(RECP1051)에서 등록 |

### 가수금 처리

> **메뉴 경로**: 수납 > [수납처리](/recp/process) > 가수금관리 (RECP1051)

초과 납부나 출처 미확인 입금은 가수금으로 등록해 두고, 이후 확인되면 해당 세대 수납에 적용합니다.

::: warning 30일 이내 처리
자동수납 오류 건은 **30일 이내에 처리**해야 합니다. 기한을 넘기면 별도의 정산 절차가 필요합니다.
:::

## 4단계: 수납 전표 연계 (3분)

> **메뉴 경로**: 수납 > [수납처리](/recp/process) > 수납전표연계 (RECP1023)

수납 내역을 회계 전표로 자동 생성합니다.

### 절차

1. 수납전표연계 화면 진입
2. **수납일자** 선택
3. **전표** 버튼 클릭 → 회계 전표 자동 생성
4. 생성된 전표 확인

### 출력 양식

- **복식전표**: 차변/대변 상세 내역
- **일계표**: 일별 수납 집계
- **수납대장**: 세대별 수납 내역

::: warning 주의
수납일자가 등록되어 있어야 전표 연계가 가능합니다. 또한, 수납을 취소하면 해당 전표도 영향을 받으므로 주의하세요.
:::

## 5단계: 수납 결과 확인 (2분)

### 일별 확인

> **메뉴 경로**: 수납 > [수납조회](/recp/inquiry) > 수납일보 (RECP5030)

날짜를 더블클릭하면 해당 일자의 수납 합계를 확인할 수 있습니다.

### 월별 집계

> **메뉴 경로**: 수납 > [수납조회](/recp/inquiry) > 수납집계표 (RECP5060)

1개월 이상의 기간을 설정하여 전체 수납 현황을 집계합니다.

### 최종 체크리스트

- [ ] 자동이체 수납 건수와 금액이 은행 결과와 일치하는가?
- [ ] 오류 건이 모두 처리되었는가?
- [ ] 가수금으로 남아 있는 미확인 입금이 없는가?
- [ ] 수납 전표가 정상 생성되었는가?
- [ ] 수납 합계와 통장 입금 합계가 일치하는가?

---

## 수납일자/은행 변경이 필요할 때

> **메뉴 경로**: 수납 > [수납처리](/recp/process) > 수납일자(은행)변경 (RECP1030)

잘못 입력된 수납일자나 은행 정보를 변경할 수 있습니다.

1. 원래 수납일자 입력
2. 변경할 일자 또는 은행 선택
3. 변경 실행

## 자주 발생하는 문제

### 자동이체 결과가 반영이 안 돼요

**원인**: 파일 형식 오류 또는 업로드 경로 문제
**해결**: 은행에서 받은 원본 파일이 정확한 형식인지 확인하고, 다시 업로드하세요.

### 이미 수납 처리한 건을 다시 수납했어요 (이중수납)

**원인**: 자동이체 + 개별 납부가 동시에 처리된 경우
**해결**: [자동이체수납](/recp/auto-transfer) > 월별이중수납 메뉴에서 확인하고, 수납취소로 중복 건을 제거하세요.

### 수납 전표 연계가 안 돼요

**원인**: 수납일자가 미등록 상태이거나, 이미 연계 처리된 건
**해결**: 수납일자를 확인하고, 전표관리(ACCT2010)에서 해당 건이 이미 있는지 확인하세요.
