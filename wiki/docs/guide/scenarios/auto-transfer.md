# 자동이체 관리

> **소요 시간**: 약 15분 | **난이도**: ★★☆ | **주기**: 매월

은행 자동이체를 통해 관리비를 수납하는 세대를 등록·관리하고, 매월 이체 결과 파일을 수신하여 수납 처리까지 완료하는 프로세스입니다.

## 전체 흐름

```
1. 자동이체 세대 등록/변경 확인  (3분)  → 신규·해지 세대 정보 업데이트
2. 자동이체 출금일 확인          (1분)  → 은행 출금 예정일 확인
3. 이체 결과 파일 수신           (2분)  → 은행에서 파일 수신
4. 자동이체 수납 처리            (5분)  → 결과 파일로 일괄 수납 처리
5. 오류 건 처리                 (3분)  → 이체 실패·이중수납 해소
6. 자동납부 신청자 확인          (1분)  → 신청 현황 최종 점검
```

---

## 사전 준비

### 자동이체 세대 등록 (최초 또는 변경 시)

> **메뉴 경로**: 부과 > [기초정보](/impo/basic) > 자동이체세대등록 (IMPO1060)

자동이체를 신청한 세대의 계좌 정보를 등록합니다. 신규 신청, 계좌 변경, 해지 시마다 갱신합니다.

### 등록 항목

| 항목 | 설명 |
|------|------|
| 동·호수 | 해당 세대 |
| 은행 | 출금 은행 |
| 계좌번호 | 출금 계좌 |
| 출금일 | 매월 자동 출금되는 날짜 |
| 적용 시작일 | 자동이체 시작 월 |

::: tip 세대에서 신청서를 받은 즉시 등록
자동이체 신청서를 받은 당일에 등록하세요. 누락 시 해당 월 자동이체에서 빠질 수 있습니다.
:::

---

## 1단계: 자동이체 세대 등록/변경 확인 (3분)

> **메뉴 경로**: 수납 > [자동이체수납](/recp/auto-transfer) > 자동납부신청자조회

이번 달 이체 실행 전에 등록 현황을 최종 점검합니다.

### 체크리스트

- [ ] 이번 달 신규 자동이체 신청 세대가 모두 등록되었는가?
- [ ] 해지 신청 세대의 등록이 삭제 또는 중지 처리되었는가?
- [ ] 계좌 변경 세대의 정보가 갱신되었는가?
- [ ] 이체 출금일이 올바르게 설정되어 있는가?

---

## 2단계: 자동이체 출금일 확인 (1분)

은행과 협의된 **자동이체 출금일**을 확인합니다. 출금일은 단지마다 다를 수 있으며, 보통 매월 특정 일자(예: 15일, 25일)로 고정됩니다.

::: warning 출금일 전날까지 등록 완료
출금일 당일 또는 직전에 세대를 등록해도 해당 월 이체에 반영되지 않을 수 있습니다. 은행에서 요구하는 **파일 제출 마감일**을 사전에 확인하세요.
:::

---

## 3단계: 이체 결과 파일 수신 (2분)

> **메뉴 경로**: 수납 > [자동이체수납](/recp/auto-transfer) > 월별자동이체

자동이체 출금 후 은행에서 결과 파일을 수신합니다.

### 절차

1. 월별자동이체 화면 진입
2. 해당 연월 선택
3. **파일 업로드** 또는 **결과 수신** 버튼으로 은행 결과 파일 불러오기
4. 이체 결과 목록 확인:
   - **성공 건**: 정상 출금 완료
   - **실패 건**: 잔액 부족, 계좌 오류, 출금 정지 등

::: tip 은행별 수신 방법 상이
은행에 따라 결과 파일을 직접 다운로드하거나 시스템 연동으로 자동 수신합니다. 처음 사용 시 은행 담당자에게 절차를 확인하세요.
:::

---

## 4단계: 자동이체 수납 처리 (5분)

> **메뉴 경로**: 수납 > [자동이체수납](/recp/auto-transfer) > 월별자동이체

이체 성공 건을 일괄 수납 처리합니다.

### 절차

1. 이체 결과 목록에서 **성공 건** 확인
2. **일괄 수납처리** 실행
3. 수납 처리 결과 확인

### 기타 수납 처리

> **메뉴 경로**: 수납 > [자동이체수납](/recp/auto-transfer) > 월별기타수납

자동이체 외 계좌이체, 현금 등으로 수납된 건을 추가로 처리합니다.

---

## 5단계: 오류 건 처리 (3분)

### 이체 실패 건 처리

> **메뉴 경로**: 수납 > [수납처리](/recp/process) (RECP1010)

이체 실패 세대는 자동으로 수납 처리가 되지 않으므로 **개별 수납 처리**가 필요합니다.

1. 이체 실패 목록 확인 (실패 사유 포함)
2. 세대에 개별 통보 (전화, 문자 등)
3. 납부 완료 후 수납처리(RECP1010)에서 수동 수납

| 실패 사유 | 조치 |
|-----------|------|
| 잔액 부족 | 세대 통보 후 개별 납부 유도 |
| 계좌 오류 | 계좌번호 재확인 후 자동이체세대등록 수정 |
| 출금 정지 | 세대 통보 및 해지 또는 계좌 변경 안내 |

### 이중수납 확인 및 처리

> **메뉴 경로**: 수납 > [자동이체수납](/recp/auto-transfer) > 월별이중수납

동일 세대에서 자동이체와 다른 방법으로 중복 납부된 건을 확인하고 처리합니다.

1. 월별이중수납 화면에서 이중수납 세대 조회
2. 이중 출금 여부 확인
3. 환불 또는 다음 달 차감 처리

::: danger 이중수납 즉시 처리
이중수납을 방치하면 세대 민원으로 이어집니다. 발견 즉시 해당 세대에 통보하고 환불 또는 차월 차감 방법을 협의하세요.
:::

---

## 6단계: 자동납부 신청자 확인 (1분)

> **메뉴 경로**: 수납 > [자동이체수납](/recp/auto-transfer) > 자동납부신청자조회

이번 달 처리 결과를 기준으로 자동납부 신청 현황을 최종 확인합니다.

### 확인 포인트

- [ ] 자동납부 신청 세대 수가 예상과 일치하는가?
- [ ] 이번 달 신규 등록 세대가 정상 반영되었는가?
- [ ] 해지 세대가 목록에서 제외되었는가?

## 자주 발생하는 문제

### 특정 세대가 자동이체 목록에 없어요

**원인**: 자동이체세대등록(IMPO1060)에 해당 세대가 등록되지 않았거나, 적용 시작일이 이번 달 이후로 설정된 경우입니다.
**해결**: 자동이체세대등록 화면에서 해당 세대를 검색하여 등록 여부와 적용 시작일을 확인하세요.

### 이체 결과 파일이 안 올라가요

**원인**: 파일 형식이 맞지 않거나, 은행에서 지정한 파일 형식과 다른 경우입니다.
**해결**: 은행에서 제공한 파일의 형식(확장자, 인코딩)을 확인하고, 시스템에서 지원하는 형식과 맞는지 비교하세요. 담당자에게 문의가 필요할 수 있습니다.

### 이체 성공인데 수납이 안 되어 있어요

**원인**: 일괄 수납처리를 실행하지 않은 경우입니다.
**해결**: 월별자동이체 화면에서 결과 파일 확인 후 **일괄 수납처리** 버튼을 클릭하세요. 파일 수신과 수납 처리는 별개 단계입니다.

### 이중수납이 발생했어요

**원인**: 세대에서 자동이체 출금 전에 창구·앱 등으로 이미 납부한 경우입니다.
**해결**: 월별이중수납 화면에서 해당 건을 확인하고, 세대와 협의하여 환불 또는 다음 달 차감으로 처리하세요.
