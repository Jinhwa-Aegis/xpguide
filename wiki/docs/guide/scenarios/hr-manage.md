# 인사 관리 (입퇴사)

> **소요 시간**: 건당 약 10분 | **난이도**: ★★☆ | **주기**: 수시 (입퇴사 발생 시)

관리사무소 직원의 입사 등록, 근태 관리, 퇴사 처리, 각종 증명서 발급까지 인사 전반을 처리하는 프로세스입니다.

## 전체 흐름

### 입사 처리

```
1. 인사기본정보 등록    (7분)  → 사원번호·성명·주민번호·부서·직급·입사일 입력
2. 급여기본정보 연동    (3분)  → 급여 구조 설정
3. 근태 관리 시작       (수시) → 연가·무급·조퇴·결근 입력
```

### 퇴사 처리

```
1. 퇴사일 입력         (2분)  → 인사기본정보에서 퇴사일 등록
2. 퇴직정산 연계       (별도) → 퇴직금 정산 프로세스 진행
3. 퇴직증명서 발급     (2분)  → 증명서출력 메뉴에서 출력
```

### 일상 업무

```
1. 근태관리 입력       (수시) → 근태 발생 시 즉시 입력
2. 입퇴사자현황 조회   (2분)  → 기간별 입퇴사 현황 확인
3. 인사기록카드 출력   (2분)  → 인사 이력 전체 조회
```

---

## 입사 처리

### 1단계: 인사기본정보 등록 (7분)

> **메뉴 경로**: 인사/급여 > [인사관리](/pay/hr) > 인사기본정보 (PAY2010)

직원의 기본 인사 정보를 시스템에 등록합니다.

#### 절차

1. 인사기본정보 화면 진입
2. **신규** 버튼 클릭
3. 필수 항목 입력 (별표 `*` 표시 항목):

| 항목 | 입력 방법 | 비고 |
|------|-----------|------|
| **사원번호** | 직접 입력 | 0001~1000 범위 |
| **성명** | 직접 입력 | |
| **주민번호** | 직접 입력 | 입력 시 생년월일 자동 반영 |
| **부서** | 인사코드에서 선택 | |
| **직급** | 인사코드에서 선택 | |
| **직종** | 인사코드에서 선택 | |
| **입사일** | 날짜 입력 | |

4. 연락처, 주소, 학력 등 추가 정보 입력
5. **저장** 클릭

#### 체크리스트

- [ ] 사원번호가 중복되지 않는가?
- [ ] 주민번호가 정확한가? (생년월일 자동 반영 확인)
- [ ] 부서·직급·직종이 인사코드에 등록된 값인가?
- [ ] 입사일이 정확한가?
- [ ] 별표(*) 필수 항목이 모두 입력되었는가?

::: tip 인사코드 확인
부서, 직급, 직종은 인사코드에 등록된 항목만 선택할 수 있습니다. 새로운 부서나 직급이 필요하다면 먼저 인사코드를 등록하세요.
:::

### 2단계: 급여기본정보 연동 등록 (3분)

> **메뉴 경로**: 인사/급여 > [급여관리](/pay/salary) > 급여기본정보 (PAY3010)

입사 등록 후 급여 계산에 필요한 급여 구조를 설정합니다.

1. 급여기본정보 화면 진입
2. 신규 등록된 직원 선택
3. 기본급, 각종 수당 입력
4. 4대보험 정보 입력 (건강보험, 국민연금, 고용보험)
5. **저장** 클릭

::: danger 입사 즉시 등록
급여기본정보에 등록되지 않은 직원은 급여 계산에서 **자동 제외**됩니다. 입사 즉시 등록하세요.
:::

상세 절차는 [급여 처리](/guide/scenarios/payroll) 시나리오를 참고하세요.

## 퇴사 처리

### 1단계: 퇴사일 입력 (2분)

> **메뉴 경로**: 인사/급여 > [인사관리](/pay/hr) > 인사기본정보 (PAY2010)

1. 퇴사 직원 검색 후 선택
2. **퇴사일** 입력
3. 퇴사 사유 입력
4. **저장** 클릭

::: warning 퇴사일 입력 후 급여·퇴직 연계
퇴사일이 입력되면 해당 월 급여는 일할 계산됩니다. 퇴직정산은 별도 메뉴에서 진행해야 합니다.
:::

### 2단계: 퇴직정산 연계

퇴직금 계산과 퇴직소득세 처리는 별도 프로세스입니다.

상세 절차는 [퇴직정산 처리](/guide/scenarios/retirement) 시나리오를 참고하세요.

### 3단계: 퇴직증명서 발급 (2분)

> **메뉴 경로**: 인사/급여 > [인사관리](/pay/hr) > 증명서출력 (PAY2140)

1. 증명서출력 화면 진입
2. 해당 직원 선택
3. **퇴직증명서** 선택
4. 출력 실행

## 일상 업무

### 근태관리

> **메뉴 경로**: 인사/급여 > [인사관리](/pay/hr) > 근태관리 (PAY2020)

직원의 연가, 무급휴가, 조퇴, 결근 등 근태 사항을 입력합니다.

#### 절차

1. 근태관리 화면 진입
2. 해당 직원 및 귀속 연월 선택
3. 근태 유형 선택:

| 근태 유형 | 설명 |
|-----------|------|
| **연가** | 유급 연차 휴가 |
| **무급** | 무급 휴가 |
| **조퇴** | 정규 퇴근 전 퇴근 |
| **결근** | 무단 결근 |

4. 날짜 및 시간 입력
5. **저장** 클릭

::: warning 급여 계산 전 근태 입력 완료
근태 데이터는 **급여 계산 전**에 모두 입력해야 정확한 급여가 산정됩니다. 급여 계산 후 근태 수정 시 재계산이 필요합니다.
:::

### 입퇴사자현황 조회

> **메뉴 경로**: 인사/급여 > [인사관리](/pay/hr) > 입퇴사자현황 (PAY2110)

기간별 입사자·퇴사자 현황을 조회합니다.

1. 조회 기간 설정
2. 입사/퇴사 구분 선택 (전체/입사/퇴사)
3. **조회** 클릭

### 인사기록카드

> **메뉴 경로**: 인사/급여 > [인사관리](/pay/hr) > 인사기록카드 (PAY2130)

직원의 인사 이력(입사, 부서 이동, 승진, 퇴사 등)을 전체적으로 확인합니다.

1. 해당 직원 선택
2. **조회** 클릭
3. 전체 이력 확인
4. 필요 시 **출력** 클릭

### 증명서 출력

> **메뉴 경로**: 인사/급여 > [인사관리](/pay/hr) > 증명서출력 (PAY2140)

| 증명서 종류 | 용도 |
|-------------|------|
| **재직증명서** | 재직 사실 증명 |
| **경력증명서** | 근무 기간·부서·직급 증명 |
| **퇴직증명서** | 퇴직 사실 및 사유 증명 |

## 자주 발생하는 문제

### 사원번호가 중복되어 등록이 안 돼요

**원인**: 이미 사용 중인 사원번호 입력
**해결**: 입퇴사자현황(PAY2110) 또는 인사기본정보 목록에서 사용 중인 번호를 확인하고 다른 번호를 입력하세요.

### 주민번호 입력 후 생년월일이 자동 입력되지 않아요

**원인**: 주민번호 형식 오류 (하이픈 포함 등)
**해결**: 주민번호를 하이픈(-) 없이 13자리 숫자만 입력하세요.

### 근태 입력 후 급여 계산에 반영이 안 돼요

**원인**: 근태 입력 전에 이미 급여 계산이 완료된 경우
**해결**: 급여 계산을 취소하고, 근태 정보를 정확히 입력한 후 다시 계산하세요.

### 부서·직급을 선택할 수 없어요

**원인**: 인사코드에 해당 항목이 등록되지 않은 경우
**해결**: [공통정보관리](/pay/common)에서 인사코드를 먼저 등록하세요.
